Answers and Explanations (concise)

Q1 — What is MVC and how does CodeIgniter implement it?

MVC (Model-View-Controller) separates concerns: Model handles data, View handles presentation, Controller handles request flow. CI4 stores controllers in app/Controllers, models in app/Models, and views in app/Views. Routes map URIs to controller methods via app/Config/Routes.php.

Q2 — Difference between GET and POST

GET retrieves data (idempotent), parameters in URL. Use for reads. POST submits data to change state and places data in the request body. Use for creates/updates.

Q3 — Why server-side validation is required

Client-side validation improves UX but can be bypassed. Server-side validation enforces business rules and prevents malicious input.

Q4 — Localization in CI4

Localization uses language files under app/Language/{locale}. Load lines via the Language service or lang() helper. The project includes a lang:sync CLI command to copy keys between locales.

Q5 — How to run the app and tests

cd /d c:\xampp\htdocs\CI4Project
php spark serve --port 8080
Visit: http://localhost:8080
Run tests: vendor\\bin\\phpunit
